# Stepper Motor Demo for PIC16F18326

This example targets the PIC16F18326 and can be opened directly inside MPLAB X.  
It drives a stepper motor driver that uses three digital signals:

| Pin | Function | Notes |
| --- | --- | --- |
| RA0 | `STEP` | Pulled high briefly for every step |
| RA1 | `DIR` | `1` = clockwise, `0` = counter‑clockwise |
| RA2 | `STALL_DIAG` | Input from the driver’s stall-detection pin |
| RA4 | `AS5600 SDA` | Bit-banged I²C for the angle sensor |
| RA5 | `AS5600 SCL` | Bit-banged I²C for the angle sensor |
| RC0 | `I2C SCL` | Assigned through PPS to MSSP1 |
| RC1 | `I2C SDA` | Assigned through PPS to MSSP1 |
| RC2 | `WS2812 DATA` | Drives a chain of 34 LEDs |
| RC3 | `ADC_IN` | Analog input sampled via ADC (exposed over I²C) |
| RC4 | `SERVO` | 50 Hz pulse generated by CCP1/CCP2 |
| RC5 | `ENABLE` | Active-low in this demo (change the defines if your driver differs) |

## What the Firmware Does

1. Configures the internal oscillator at 32 MHz.
2. Sets RA0, RA1, and RC5 as digital outputs.
3. Uses Timer2 interrupts to generate precise STEP pulses.
4. Applies a trapezoidal speed ramp (constant acceleration → cruise → constant deceleration) for smoother torque.
5. Monitors RA2 for stall diagnostics and stops motion if a fault is reported.
6. Exposes an interrupt-driven I²C slave (address `0x22`) so a master can send commands or read status bytes.
7. Animates a strip of 34 WS2812B LEDs on RC2 with a directional ripple while the motor is running.
8. Drives a hobby servo on RC4 (50 Hz pulse) so you can position an auxiliary joint.
9. Polls an AS5600 magnetic encoder on RA4/RA5 (bit-banged I²C) and exposes its readings.
10. Motor angle is controlled exclusively via I²C commands—no automatic movement.

## Timing Numbers

- `SLOW_DELAY_TICKS` and `FAST_DELAY_TICKS` control the lowest and highest speeds.
- Timer2 runs from `Fosc/4` with a 1:64 prescaler, so one timer tick equals **8 µs**.
- The current settings produce a pace between roughly **156** and **781** full steps per second (assuming 200 steps/rev). Adjust the constants if you need different speeds.

## I²C Command Framework

- Default 7-bit slave address: `0x22`.
- Simple write commands:
  - `0x01 <LSB> <MSB>`: move to target angle in degrees. Angle is sent as a 16-bit value in hundredths of a degree (LSB first). Example: `0x01 0x28 0x23` moves to 90.00° (9000 in decimal = 0x2328, sent as LSB=0x28, MSB=0x23).
  - `0x02`: immediately disable the driver and stop motion.
  - `0x03 <angle>`: point the servo on RC4 to `<angle>` degrees (0‑180).
  - `0x04 <addr>`: update the AS5600 I²C address (stored in EEPROM).
  - `0x05 <LSB> <MSB>`: update the AS5600 offset (hundredths of a degree, stored in EEPROM).
  - `0x06`: request the settings packet (address + offset + kinematics) on the next host read.
  - `0x07`: request the ADC packet (RC3 raw counts + millivolts) on the next host read.
  - `0x08 <state>`: disable the servo driver and drive RC4 as a relay output (`0`=off, `1`=on).
  - `0x09 <B0> <B1> <B2> <B3>`: set kinematics d (link offset) in hundredths of mm (32-bit signed, LSB first, stored in EEPROM).
  - `0x0A <B0> <B1> <B2> <B3>`: set kinematics a (link length) in hundredths of mm (32-bit signed, LSB first, stored in EEPROM).
  - `0x0B <B0> <B1> <B2> <B3>`: set kinematics theta (joint angle) in hundredths of degree (32-bit signed, LSB first, stored in EEPROM).
  - `0x0C <B0> <B1> <B2> <B3>`: set kinematics alpha (link twist) in hundredths of degree (32-bit signed, LSB first, stored in EEPROM).
- Simple read packets:
  - Status packet (default, 8 bytes):
    1. `is_moving` flag (1 = running, 0 = idle)
    2. `stall_detected` flag
    3-6. `current_step_position` (LSB first, signed 32-bit)
    7-8. AS5600 angle in hundredths of a degree (LSB first, unsigned 16-bit)
  - Settings packet (returned once after command `0x06`, 20 bytes):
    1. `0xAA` marker
    2. Stored AS5600 I²C address
    3-4. Stored offset in hundredths of a degree (LSB first, signed 16-bit)
    5-8. Kinematics d (link offset) in hundredths of mm (LSB first, signed 32-bit)
    9-12. Kinematics a (link length) in hundredths of mm (LSB first, signed 32-bit)
    13-16. Kinematics theta (joint angle) in hundredths of degree (LSB first, signed 32-bit)
    17-20. Kinematics alpha (link twist) in hundredths of degree (LSB first, signed 32-bit)
  - ADC packet (returned once after command `0x07`, 5 bytes):
     1. `0xAB` marker
     2-3. Latest RC3 ADC raw counts (LSB first)
     4-5. RC3 voltage estimate in millivolts (LSB first)
- Use these as a starting point—expand the command set or status packet to suit your application.

## LED Ripple Animation

- Connect the DIN pin of the WS2812B strip to RC2 (DIN), share ground and +5 V.
- The firmware maintains a 34‑LED buffer and bit-bangs the data line at 32 MHz (interrupts are briefly disabled only while sending LED frames).
- When the stepper starts moving, a soft ripple appears and marches in the same direction as the motor rotation. The ripple halts and LEDs turn off automatically when motion stops or a stall is detected.
- Tweak colors or speeds inside `led_ripple.c` (look for `leds_apply_ripple_frame()` and `LED_UPDATE_TICKS`).

## Servo Output (RC4)

- CCP1 schedules the 20 ms frame and CCP2 schedules the 1–2.5 ms pulse width; the firmware toggles RC4 high/low inside the interrupts.
- Change the servo angle either in code (`ServoPwm_SetAngleDegrees`) or via I²C command `0x03 <angle>` where `<angle>` ranges from 0 to 180.
- The pulse width range defaults to 1000–2500 µs. Adjust `SERVO_MIN_US` / `SERVO_MAX_US` in `servo_pwm.c` if your servo needs a different span.
- If you need RC4 as a simple relay output, send I²C command `0x08 <state>`—this turns off the servo PWM and drives RC4 as a steady digital level until another servo command is issued.

## AS5600 Angle Sensor (RA4/RA5)

- The magnetic encoder connects to RA4 (SDA) and RA5 (SCL). Because MSSP1 is already used for the host link, `as5600_sensor.c` bit-bangs the bus.
- `As5600_Service()` polls the ANGLE registers, converts the 0‑4095 raw counts into real degrees (0‑360), and caches both values so other modules (and the host) can read them without blocking.
- The cached degrees are appended to the I²C status packet (bytes 7–8) in hundredths of a degree. Adjust the polling strategy or filtering inside `as5600_sensor.c` if you need different behavior.

## RC3 Analog Input

- RC3 is configured as an analog input and sampled once per main loop by `adc_rc3.c`.
- Use I²C command `0x07` to fetch the most recent reading (both raw counts and millivolts).
- Internally the ADC assumes VDD = 5 V; adjust `VREF_MV` in `adc_rc3.c` if you run at a different reference.

## Kinematics Constants (Denavit-Hartenberg Parameters)

- The firmware stores Denavit-Hartenberg (DH) parameters for robot arm kinematics in EEPROM:
  - **d** (link offset): Distance along previous z-axis, stored in hundredths of mm (0.01mm resolution)
  - **a** (link length): Distance along x-axis, stored in hundredths of mm (0.01mm resolution)
  - **theta** (joint angle): Rotation about z-axis, stored in hundredths of degree (0.01° resolution)
  - **alpha** (link twist): Rotation about x-axis, stored in hundredths of degree (0.01° resolution)
- Use I²C commands `0x09`-`0x0C` to set each parameter (values are saved to EEPROM immediately).
- Use I²C command `0x06` to read all settings including kinematics constants.
- These values are stored persistently and loaded on startup, allowing calibration of the joint's kinematic model.

## Next Steps for Customization

- Set `STEPS_PER_REV` and `MICROSTEPS` to match your motor driver.
- Fine-tune the motion profile by editing `STEPPER_MIN_SPEED_SPS`, `STEPPER_MAX_SPEED_SPS`, and `STEPPER_ACCEL_SPS2` in `motor_control.c`.
- Replace the simple `while` loop with your own command source (buttons, serial, etc.).
- The defaults are conservative for a NEMA 17 on a TMC2226 at 16× microstepping; feel free to raise the max speed/acceleration once you validate torque margins.

Everything in `main.c` is heavily commented so you can extend it at your own pace. Have fun experimenting!
