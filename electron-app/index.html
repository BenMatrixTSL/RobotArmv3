G-Code Cheat Sheet (Supported Commands)

The app supports a small, beginner-friendly subset of G-code. Use this list as a quick reference while you are editing your program.

G0, G1 Linear move command. Supports multiple modes:
• Stored position (by number): G1 P5 F60 → move to stored position 5 at 60 deg/s
• Stored position (by name): G1 P"Home" F60 → move to stored position named "Home"
• Joint-space moves: G1 J1=45 J2=20 F60 → move joints 1 and 2 to 45° and 20°
• Cartesian XYZ: G1 X150 Y0 Z200 F60 → move TCP to XYZ position (requires kinematics)
The P parameter can be a number (0-99) or a quoted string (position name) from the Stored Positions tab.
G28 Home all joints to 0° (uses optional F for speed in deg/s).
G28 F45 → home joints at 45 deg/s.
Joint moves Move specific joints directly using J1…J5 with angles in degrees.
J1=45 J2=-30 F60 → move J1 and J2 to 45° and −30° at 60 deg/s (scaled so they finish together).
M0, M1 Pause the program.
M0 or M1 → pause for 2000 ms (2 s).
M0 P5000 → pause for 5000 ms (5 s).
M204 Set servo acceleration (0–254, in units of 100 step/s²).
M204 A80 → set acceleration for all joints to 80.
M204 J3 A100 → set acceleration for joint 3 only.
M30, M2 Program end marker. Execution stops when these are reached.
M30 or M2.
Comments Use ; or ( ... ) for comments:
G1 X10 Y0 Z0 ; move to start
(This is a comment line)

RAPID Editor

RAPID Cheat Sheet (Supported Subset)

This is not a full RAPID interpreter. It only supports a tiny subset that maps cleanly to your 5‑axis arm.

Comments Start a line with ! for a comment.
! This is a comment
MoveJ Joint move using 5 joint angles in degrees.
MoveJ [[J1,J2,J3,J4,J5]];
Example: MoveJ [[0,10,20,0,0]]; → move to J1=0°, J2=10°, J3=20°, J4=0°, J5=0° (default speed).
MoveAbsJ Absolute joint move (same behaviour as MoveJ, just a different name).
MoveAbsJ [[0,0,30,0,0]];
MoveJOffs Joint move using offsets added to the current joint angles.
MoveJOffs [[dJ1,dJ2,dJ3,dJ4,dJ5]];
Example: MoveJOffs [[0,-5,-10,0,0]]; → subtract 5° from J2 and 10° from J3.
MoveLXYZ Cartesian move to an absolute XYZ position (in mm). Uses the kinematics solver internally.
MoveLXYZ [[X,Y,Z]];
Example: MoveLXYZ [[150,0,200]]; → move TCP to X=150 mm, Y=0 mm, Z=200 mm.
MoveLOffs Cartesian offset move relative to the current XYZ position (in mm).
MoveLOffs [[dX,dY,dZ]];
Example: MoveLOffs [[0,0,50]]; → move the TCP 50 mm upwards from where it is now.
WaitTime Pause the program for a number of seconds.
WaitTime 2; → wait for 2 seconds.
SetDO Set a (simulated) digital output channel to 0 or 1.
SetDO 1, 1; → turn output 1 on.
SetDO 1, 0; → turn output 1 off.
Home Convenience command that moves all joints back to 0°.
Home;

Kinematics and Transform Matrices

Simulated Joint Angles (for Kinematics)

These angles are the same ones used by the 3D visualisation when "Use Simulated Angles" is enabled. Adjust them here to see how the transforms and XYZ positions change.

This tab shows how forward kinematics builds up from the base to the end effector. For each joint, we apply a transform Ti = Origini · Rotation(axisi, anglei), and multiply them to get the cumulative transform.

4×4 transform layout

R11 R12 R13 Tx (mm)
R21 R22 R23 Ty (mm)
R31 R32 R33 Tz (mm)
0 0 0 1

The top-left 3×3 entries Rij form the rotation matrix. The last column holds the translation in millimetres (internally stored in metres).

AS5600 Sensor Calibration

Calibrate the AS5600 sensor for a joint. The calibration routine will:
1. Move forward until hitting the mechanical end stop
2. Move backward until hitting the other mechanical end stop
3. Move to the middle position
4. Reset the AS5600 absolute angle to 0° at the center position
