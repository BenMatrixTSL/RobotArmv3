# Module 14: Project Planning & Management
## Multiple Choice Questions

**Instructions:** Choose the best answer for each question.

---

**1. Why is project planning important?**
   a) It's not important
   b) To organize work, manage time, and ensure success
   c) To make it faster
   d) To make it slower

**2. What is project scope?**
   a) What the project will and won't include
   b) The project's speed
   c) The project's cost
   d) The project's color

**3. What are functional requirements?**
   a) What the system must do
   b) How fast it must be
   c) How much it costs
   d) What color it is

**4. What is a task breakdown?**
   a) Dividing project into manageable tasks
   b) Breaking equipment
   c) Stopping work
   d) Making it faster

**5. What is a dependency?**
   a) When one task must be completed before another
   b) When tasks are independent
   c) When tasks are the same
   d) When tasks don't matter

**6. What is a critical path?**
   a) The longest sequence of dependent tasks
   b) The shortest path
   c) The easiest path
   d) The most expensive path

**7. Why track progress?**
   a) To monitor project status and identify issues
   b) It's not important
   c) To make it faster
   d) To make it slower

**8. What is risk management?**
   a) Identifying and mitigating project risks
   b) Ignoring risks
   c) Creating risks
   d) Avoiding planning

**9. Why is communication important in projects?**
   a) To keep stakeholders informed
   b) It's not important
   c) To make it faster
   d) To make it slower

**10. What is a Gantt chart?**
   a) A visual timeline of project tasks
   b) A list of tasks
   c) A budget
   d) A schedule

---

## Answer Key

1. **b)** To organize work, manage time, and ensure success
2. **a)** What the project will and won't include
3. **a)** What the system must do
4. **a)** Dividing project into manageable tasks
5. **a)** When one task must be completed before another
6. **a)** The longest sequence of dependent tasks
7. **a)** To monitor project status and identify issues
8. **a)** Identifying and mitigating project risks
9. **a)** To keep stakeholders informed
10. **a)** A visual timeline of project tasks
