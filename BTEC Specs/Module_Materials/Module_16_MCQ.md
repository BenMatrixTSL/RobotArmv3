# Module 16: Evaluation, Reflection & Communication
## Multiple Choice Questions

**Instructions:** Choose the best answer for each question.

---

**1. What is functional evaluation?**
   a) Checking if system meets requirements
   b) Checking speed only
   c) Checking cost only
   d) Checking appearance only

**2. What is performance evaluation?**
   a) Measuring speed, accuracy, repeatability, reliability
   b) Measuring cost only
   c) Measuring size only
   d) Measuring color only

**3. What is repeatability?**
   a) How consistently system returns to same position
   b) How fast it moves
   c) How much it costs
   d) How big it is

**4. What is reliability?**
   a) How consistently system works without failure
   b) How fast it moves
   c) How much it costs
   d) How big it is

**5. Why reflect on a project?**
   a) To learn from experience and improve
   b) It's not important
   c) To make it faster
   d) To make it slower

**6. What should a technical presentation include?**
   a) Technical details for technical audience
   b) Only pictures
   c) Only text
   d) Only costs

**7. What should a non-technical presentation include?**
   a) Simplified explanations for general audience
   b) Only technical details
   c) Only code
   d) Only diagrams

**8. Why identify limitations?**
   a) To be honest about what system can and can't do
   b) To hide problems
   c) To make it look better
   d) It's not important

**9. Why propose improvements?**
   a) To show how system could be enhanced
   b) To criticize only
   c) To avoid work
   d) It's not important

**10. How does this work support career progression?**
   a) Develops skills relevant to engineering employment
   b) It doesn't help
   c) Only helps with programming
   d) Only helps with mechanics

---

## Answer Key

1. **a)** Checking if system meets requirements
2. **a)** Measuring speed, accuracy, repeatability, reliability
3. **a)** How consistently system returns to same position
4. **a)** How consistently system works without failure
5. **a)** To learn from experience and improve
6. **a)** Technical details for technical audience
7. **a)** Simplified explanations for general audience
8. **a)** To be honest about what system can and can't do
9. **a)** To show how system could be enhanced
10. **a)** Develops skills relevant to engineering employment
