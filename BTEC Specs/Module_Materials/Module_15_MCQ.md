# Module 15: Technical Documentation
## Multiple Choice Questions

**Instructions:** Choose the best answer for each question.

---

**1. Why is technical documentation important?**
   a) It's not important
   b) To communicate system design and enable maintenance
   c) To make it faster
   d) To make it slower

**2. What is a system block diagram?**
   a) A visual representation of system components
   b) A list of parts
   c) A budget
   d) A schedule

**3. What is a wiring diagram?**
   a) A diagram showing electrical connections
   b) A diagram showing mechanical parts
   c) A diagram showing software
   d) A diagram showing costs

**4. What is URDF?**
   a) Unified Robot Description Format
   b) A programming language
   c) A controller
   d) A sensor

**5. What should a test plan include?**
   a) Test objectives, procedures, and cases
   b) Only test results
   c) Only test objectives
   d) Only procedures

**6. Why document test results?**
   a) To verify system meets requirements
   b) It's not important
   c) To make it faster
   d) To make it slower

**7. What is version control?**
   a) Tracking changes to documentation
   b) Controlling the robot
   c) Controlling speed
   d) Controlling cost

**8. Why review documentation?**
   a) To ensure accuracy and completeness
   b) It's not important
   c) To make it faster
   d) To make it slower

**9. What should mechanical documentation include?**
   a) Kinematics, links, joints, workspace
   b) Only software
   c) Only electrical
   d) Only costs

**10. What makes good documentation?**
   a) Clear, complete, accurate, organized
   b) Vague and incomplete
   c) Only diagrams
   d) Only text

---

## Answer Key

1. **b)** To communicate system design and enable maintenance
2. **a)** A visual representation of system components
3. **a)** A diagram showing electrical connections
4. **a)** Unified Robot Description Format
5. **a)** Test objectives, procedures, and cases
6. **a)** To verify system meets requirements
7. **a)** Tracking changes to documentation
8. **a)** To ensure accuracy and completeness
9. **a)** Kinematics, links, joints, workspace
10. **a)** Clear, complete, accurate, organized

---

## Scoring Guide

- **9-10 correct:** Excellent understanding
- **7-8 correct:** Good understanding
- **5-6 correct:** Satisfactory understanding
- **Below 5 correct:** Needs review
